Клонування репозиторію:
  • Команда: git clone "URL репозиторію"
  • Опис: Клонування репозиторію забезпечує створення копії віддаленого репозиторію на локальному комп'ютері. Використовується для отримання повної інформації про проект та його історії.
    Створення нової гілки:
  • Команда: git branch "ім'я_гілки"
  • Опис: Створення нової гілки дозволяє розробникам працювати над різними функціональностями паралельно, не впливаючи на основну гілку.
    Перехід між гілками:
  • Команда: git checkout "ім'я_гілки"
  • Опис: Перехід між гілками дозволяє змінити робочу гілку, на якій ви зараз працюєте.
    Створення коміту:
  • Команда: git commit -m "Ваш коментар до коміту"
  • Опис: Коміт фіксує зміни в робочій директорії та додає їх до історії проекту. Коментар надає короткий опис внесених змін.
    Пошук і відміна змін:
  • Команда: git status, git diff
  • Опис: git status показує статус змінених файлів, а git diff виводить внесені зміни. Дозволяють перевірити і контролювати стан робочої директорії перед комітом.
    Злиття гілок:
  • Команда: git merge "ім'я_гілки"
  • Опис: Злиття гілок дозволяє об'єднати зміни з однієї гілки в іншу. Може бути використано для об'єднання функціональностей розроблених в різних гілках.
    Відправлення змін на віддалений репозиторій (Push):
  • Команда: git push "віддалений_репозиторій" "ім'я_гілки"
  • Опис: Відправлення змін на віддалений репозиторій оновлює віддалену гілку з власними локальними змінами
    Оновлення локального репозиторію (Pull):
  • Команда: git pull "віддалений_репозиторій" "ім'я_гілки"
  • Опис: Витягує зміни з віддаленого репозиторію та об'єднує їх з локальною гілкою.
